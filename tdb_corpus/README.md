# TDB Corpus — local ensemble of reference databases

This directory builds and maintains a **local corpus of CALPHAD `.tdb`
files** drawn from openly-available sources, with tooling to query
Brown's **TDBDB** (the index-of-databases at
[avdwgroup.engin.brown.edu](https://avdwgroup.engin.brown.edu)) for
per-system citation lists.

The corpus is the substrate for downstream **ensemble Stage-3
scoring** (Layer 1 of the consensus-targets discussion): instead of
ranking candidate TDBs against one reference, rank against the
mean ± uncertainty across N curated references.

## How to use it (workflow)

### One-time setup

```bash
# 1. Optional: pull the open-licensed TDBs into the corpus.
python3 tdb_corpus/tdbdb_fetch_open.py --download

# 2. For paywalled / by-permission TDBs:
#    fetch each manually via your library proxy (browser, not script —
#    see "Licensing posture" below), then drop the .TDB into corpus/.

# 3. Index everything that's now in corpus/.
python3 tdb_corpus/corpus_ingest.py
# -> writes corpus/_manifest.yaml
```

### Adding a TDB manually

```bash
# 1. Drop the file:
cp ~/Downloads/Hallstedt2017_AlMg.TDB tdb_corpus/corpus/

# 2. (Optional but recommended) write a sidecar citation file with the
#    same basename so the manifest tracks attribution:
cat > tdb_corpus/corpus/Hallstedt2017_AlMg.citation.yaml <<'EOF'
title: "Thermodynamic assessment of the Al–Mg system, revised"
authors: ["B. Hallstedt"]
year: 2017
doi: "10.1016/j.calphad.2017.05.004"
source_url: "https://doi.org/10.1016/j.calphad.2017.05.004"
license: "CC-BY-4.0"   # or "publisher-paywalled", "by-author-permission", etc.
notes: "Replaces Hallstedt 2007 binary."
EOF

# 3. Re-index.
python3 tdb_corpus/corpus_ingest.py
```

### Querying TDBDB for new citations

```bash
# Once the API spec lands in api_spec/ and tdbdb_query.py is wired up:
python3 tdb_corpus/tdbdb_query.py --elements Co,Cr,Ni
# -> prints citation list; does NOT download files.
```

## What you need to do (the API piece)

Browse to **https://avdwgroup.engin.brown.edu/help.html** (Brown
WAF blocks our sandbox IP) and save the page contents into
`api_spec/help.html` (or just paste the text into
`api_spec/help.txt`). Anything that documents:

- the base URL convention,
- query string parameters (elements, phases, etc.),
- response format (JSON / HTML / XML),
- authentication / rate limits,
- citation requirements,

is enough for me to fill in `tdbdb_query.py`. A sample response from
one query (e.g. Co-Cr) would also help — paste it into
`api_spec/sample_response.txt`.

## Licensing posture (read this before you scrape anything)

The reasons for keeping the fetcher conservative:

- TDBDB itself is **an index** (766 entries / 528 publications) — it
  points at TDB files hosted elsewhere (publisher SI sections, NIMS,
  NIST MDR, author personal pages). It does not host the bytes.
- Many indexed sources are **paywalled** (Elsevier Calphad SI,
  Springer, etc.). Their TOS prohibit scripted bulk download even
  through authorized institutional IPs.
- Even *open-access* sources have specific licenses: CC-BY needs
  attribution and license preservation; some "free to use" personal
  pages disallow redistribution.

The fetcher therefore only auto-downloads from sources whose license
is **explicitly redistribution-friendly** (CC-BY, MIT, etc.). For
everything else it prints the citation + URL and asks you to fetch
manually via your library — that's the only TOS-clean path.

`.gitignore` in `corpus/` keeps `.TDB`/`.tdb` files **out of git**
by default, so we don't accidentally re-publish paywalled databases.
Track `_manifest.yaml` (and `*.citation.yaml` sidecars) in git so
the corpus composition is reproducible without redistributing the
content itself.

---

## What this is *not*

- Not an attempt to be Thermo-Calc or OpenCalphad. Those are
  consumers of TDB files; we just collect them for ensemble scoring.
- Not a parser. `pycalphad.Database` reads TDBs; we use it directly.
- Not a redistribution channel for commercial / publisher-restricted
  databases. Manifest entries with `license: paywalled` are
  citations only — the bytes stay on your machine.
