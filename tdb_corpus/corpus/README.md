# corpus/ — local TDB files

This directory holds the **actual `.tdb` / `.TDB` files** that downstream
ensemble-scoring code reads. The files themselves are **not tracked in
git** (see `.gitignore`) so we don't accidentally redistribute
paywalled or per-author-permission databases through this repository.

## What to put here

- Open-licensed `.tdb` files (auto-fetched by
  `../tdbdb_fetch_open.py` or dropped in manually).
- Optional sidecar metadata: `<basename>.citation.yaml` next to each
  TDB, describing source / license / DOI. `corpus_ingest.py` picks
  these up automatically.

## What gets tracked in git

- `README.md` (this file)
- `.gitignore`
- `_manifest.yaml` — auto-generated by `../corpus_ingest.py`. Records
  every file's elements, phases, sublattice models, and citation
  metadata so the corpus composition is reproducible without
  redistributing the bytes.
- `*.citation.yaml` sidecars — text metadata only, safe to share.

## Naming convention

No hard rule, but a helpful pattern is `<FirstAuthor><Year>_<System>.TDB`,
e.g. `Hallstedt2017_AlMg.TDB`. The sidecar then matches:
`Hallstedt2017_AlMg.citation.yaml`. Multiple assessments for the same
system live side-by-side and the ensemble scorer pulls all of them in.
